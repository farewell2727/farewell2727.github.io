速通博客搭建

发表于2023-11-05|更新于2023-11-12
|阅读量:

一：写在前面

此教程不需要太多专业性相关知识储备，想加深探索可以注意一下每一个步骤里面的细节。大多数是参考现成教程，仅仅起到速成网站的作用。此教程其基本原理是使用hexo模板（网站前端），github（代替购买域名），sublimetext（网站的维护与内容编辑）。
参考教程：1.使用Hexo搭建个人博客手摸手教学(1)|前言
2.Hexo+github搭建个人博客并绑定个人域名
跟着这两个教程的步骤基本上可以完成一整套流程的搭建。（p.s.前提是电脑已经完成相关环境搭建工作！）

二：准备工具列表

node.js（java运行环境）
python（需要最新版的版本号是3.12）
pycharm社区版（用来下载python包）
sublimetext
pip（python3.9以上版本自带pip，也就是最新版python自带pip包。）
git（项目管理工具）
sidedevcar（用于网络加速，说实话想一劳永逸就去搞个v2ray，但是这个免费，就是可能安装根证书需要麻烦一点，推荐先下载这个防止有些网站打不开）
此外还需要准备一个github账号（最好用outlook邮箱注册）。
网站编写内容使用mkdown语法，该轻量级语言可以在处理大部分网站需求，以及有可以快速上手的优点等等。

注意：搭建环境的时候一般将这些东西都放在c盘也就是系统盘。如果空间不够，可以把pycharm，sublimetext，sidedeccar挪到其他盘。这一步是为了防止后续弹出其他报错。此外安装路径最好不要出现中文，所有安装进程执行标准安装即可。

三：搭建教程

第一步運行檢查

检查git版本，右键任务栏里面的win标，点击终端（管理员）。輸入以下内容。
git version
如果git安装成功，会弹出相应版本号。

检查node版本，步骤同上。（终端窗口先不用关闭）
node -v
成功的话也会弹出相应版本号。

检查npm版本，步骤同上。
npm -v
成功的话也会弹出相应版本号。

至此基本工作已经运行，至于其他软件不需要检查版本号，正确安装即可。

安装全局express模块。（同樣在剛才的管理員終端窗口運行）
npm install express -g
（到这一步要注意一点了，这里使用的终端都是在管理员权限之下的终端，右键win标会出来两个终端，一个是普通终端，另一个是管理员权限的终端，用后者来执行命令会省去很多很多麻烦。win加r快捷方式再加cmd那个应该也是普通权限的终端）

补充说明，如果出现报错请复制窗口的报错内容，然后去csdn上面找解决方法。

第二步创建github相关仓库步骤

首先注册一个邮箱，这里推荐使用微软自带的outlook邮箱（这个就算跳出这个教程，以后是也很有用的喔）或者火狐邮箱，雅虎邮箱，gmail都可。由于接码问题不建议使用国内邮箱。
打开github官网，点击sign up来进行注册，验证邮箱使用刚刚注册的邮箱。用户名随便填。这里推荐使用英文用户名。

新建一个仓库，点击加号旁边的倒三角，再点击new repository。在彈出的界面裏面，repository name後面倉庫名一定是這個格式：你的github賬戶名＋.github.io。（例：xxxxx.github.io）這個格式必須要一致。然後在description當中有兩個選項：

  • Public
  • Private
    選擇public。這樣倉庫就創建好了。

第三步生成ssh密鑰

這一步的目的是設置管理者把本地内容上傳到github上你賬戶的倉庫時使用的密鑰。
在桌面右鍵，然後點擊顯示更多選項，然後點擊Git Bash here。輸入以下内容。
ssh-keygen -t rsa -C "你的邮箱地址"
稍等一會，需要按照提示按四次回車鍵。第二次回車的時候輸入y。
测试一下密钥是否绑定成功，在窗口输入以下内容。
ssh -T git@github.com
如果成功的话会出现含有successfully的提示。
出现报错 git@github.com: Permission denied (publickey). 解决方法参阅此处

第四步创建本地blog文件夹

此处建议存放在d盘下。
d盘右键，创建一个新文件夹，命名为blog。进入这个文件夹，右键，启用git bash here。然后进行初始化hexo工作，输入以下代码。
hexo init
如果成功的话会提示info start blogging with hexo！

第五步运行本地界面

在第四步窗口里面输入以下代码：
hexo s
之后会提示info hexo is running at （本地地址）。复制这个地址，打开浏览器进入。
在git bash窗口长按ctrl+c关闭本地服务器。
